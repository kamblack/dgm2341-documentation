Asset Management

In choosing a strategy for organizing my files, I knew what because I'm working on a Windows machine, I would have to lean into two main tools: the names of the files themselves, and a system of folders. I don't have the option to tag files or otherwise mark up the metadata, at least not in a way that is uniform among all different file types.

With that said, there are two projects that are part of this class that I have to manage. This Documentation is the first one. The other is a "Use Case" or an example of an actual product as an example of real-life application.

First off, there are three different folders dedicated to working with the three main different types of files, audio, video, and images. In each there will be an ingest folder for the original files, an output folder for the final product, and any necessary folders for experimentation in between those two steps. In the evaluation folders, the file name defines the different details about the individual file and its output settings for quick comparison.

There are also folders dedicated to the Documentation and to the use case. Both the use case project and the documentation will be web pages. For me, this means that they will have an actual root document, index, as well as resource folders for images, video, and audio as necessary. Both folders will also have the necessary Git files to keep everything clean and trackable. The major difference between the use case project and the documentation is that the documentation index is split into multiple chapters, each with their own HTML. When this is published, the individual chapters will all be concatenated together into one long document.
